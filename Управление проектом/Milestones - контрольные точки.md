## Контрольные точки (Milestones)
> Контрольные точки (milestones) - это ключевые этапы проекта, где мы проверяем прогресс, достижение целей и корректируем план. Они основаны на Agile-методологии (Scrum, спринты по 2 недели), релизах (MVP, Release 2 и т.д.), User Story Map (USM) и DFD (поток данных). Каждая точка имеет дату (относительную, от старта проекта), критерии успеха, ответственных и риски.

> Проект стартует с MVP, цель - запуск устойчивого продукта за 6–12 месяцев.

### Общие правила для контрольных точек:

1. **Частота проверки:**  Еженедельно + ежемесячно (отчеты руководителю Незнанову Е.Д.).
  
2. **Критерии успеха:** Прохождение тестов (95% покрытия кода), положительный фидбек от пользователей (опросы).
   
3. **Ответственные:** Проектный менеджер (Веревкина Д.А.) координирует, команда (разработчики, дизайнеры) выполняет, руководитель (Незнанов Е.Д.) одобряет.
 
4. **Риски и корректировка:** Если точка не достигнута (например, задержка в AI), перенос на 1–2 недели, анализ причин (SWOT).
 
5. **Метрики:**  коэфицент удержания, NPS (лояльность пользователя), количество пользователей, рост дохода.

## Список контрольных точек:

### 1. Завершение MVP (Milestone 1: Базовый продукт)
- **Дата:** 26 мая.
- **Описание:** Запуск минимально жизнеспособного продукта с базовыми фичами из USM (регистрация, установка цели, отслеживание прогресса, достижения).
- **Критерии успеха:** 1000+ активных пользователей, все тесты пройдены.
- **Ответственные:** Команда (разработка MVP) + проверка (Грохульский М.С, Незнанов Е.Д).
- **Риски:** Ошибки в AI (недоверие 21,8% пользователей) - решение: перепроверка данных.
- **Результат:** Фидбек для Release 2.

<br>

### Release 2: Расширенные фичи (Milestone 2: Улучшения)
- **Дата:** Через 3–4 месяца (после MVP + 4 спринта).
- **Описание:** Добавление архива задач, магазина, уведомлений, уровней, рейтинга. Связь с USM: микрозадачи, редакция этапов, индивидуальные настройки.
- **Критерии успеха:** Рост пользователей на 20–50%.
- **Ответственные:** Команда (добавление фич) + проверка (Грохульский М.С, Незнанов Е.Д).
- **Риски:** Перегруженность интерфейса (как у конкурентов) - решение: A/B-тестирование.
- **Результат:** Монетизация через премиум (особые значки, ускорение).

<br>

### Release 3: Сообщества и аналитика (Milestone 3: Социализация)
- **Дата:** Через 5–6 месяцев (после Release 2 + 4 спринта).
- **Описание:** Добавление сообществ, челленджей, экспорта, оффлайн-доступа, анализа AI. Связь с USM: комментирование, прогресс друзей, туториал. Проверка DFD: поток данных между пользователями.
- **Критерии успеха:** 10к+ пользователей, участие в челленджах 30% аудитории, NPS >60.
- **Ответственные:** Команда (интеграция AI и сообществ), + проверка (Грохульский М.С, Незнанов Е.Д).
- **Риски:** Проблемы с приватностью (GDPR) - решение: шифрование.
- **Результат:** Расширение на блогеров и партнеров.

<br>

### Release 4: Группы и масштабирование (Milestone 4: Полный продукт)
- **Дата:** Через 7–9 месяцев (после Release 3 + 4 спринта).
- **Описание:** Добавление групп для совместных целей, создания сообществ. Связь с USM: группы, поиск, экспорт истории. Проверка DFD: масштабирование на 10k+ пользователей.
- **Критерии успеха:** стабильный доход 200 тыс. руб./год, устойчивый рост 20–50% ежемесячно.
- **Ответственные:** Команда (масштабирование) + проверка (Грохульский М.С, Незнанов Е.Д).
- **Риски:** Технические сбои при росте - решение: нагрузочное тестирование.
- **Результат:** Готовый продукт для долгосрочного использования.

<br>

  ### Полный запуск (Milestone 5: Завершение фазы)
- **Дата:** Через 10–12 месяцев.
- **Описание:** Финальная оценка: все релизы запущены, фидбек собран, метрики достигнуты. 
- **Критерии успеха:** Ёмкость рынка 3–5 млрд руб. (рост), портфолио для команды, социальная выгода (помощь молодежи).
- **Ответственные:** Команда (отчет) + проверка (Грохульский М.С, Незнанов Е.Д)
- **Результат:** Переход к поддержке.
